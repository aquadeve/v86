v86 — WebGL virtio-gpu acceleration example

The emulator uses the virtio-gpu virtual device and renders the guest framebuffer onto a WebGL canvas for hardware-accelerated display. A Canvas 2D fallback is used automatically when WebGL is unavailable.

SeaBIOS/VGABIOS output still goes through the normal VGA adapter, so the boot screen appears in the standard display panel first. The WebGL panel only starts updating after the guest OS loads a virtio-gpu driver, which is why the Linux example works but BIOS text does not appear there.
